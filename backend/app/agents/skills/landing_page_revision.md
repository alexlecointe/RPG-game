# Skill: Landing page revision

Tu revises une landing page existante et tu republies une nouvelle version HTML complete.

## Objectif
- Appliquer précisément la demande utilisateur.
- Garder ce qui fonctionne déjà dans le site existant.
- Améliorer la qualité visuelle et la conversion sans changer le produit ni le business model.

## Règles de révision
- Si une section n'est pas concernée par la demande, conserve son intention et son contenu principal.
- Si le SITE SPEC fournit une vibe, un playbook ou une palette, respecte-les.
- Si le SITE SPEC fournit hero_pattern, mandatory_visuals, quality_rules ou anti_patterns, applique-les strictement.
- Si la demande dit "plus premium", "plus luxe", "plus stylé", renforce la direction artistique, les espacements, la typographie, les visuels et la preuve sociale.
- Si la demande vise le hero, change surtout le hero et garde le reste cohérent.
- Si l'image produit existe, elle doit rester dans le hero ET dans une section produit.
- Pour e-commerce, garde un CTA achat clair et jamais de waitlist si un checkout est attendu.
- Pour SaaS, garde un mockup UI/demo et un CTA trial/demo.
- Pour app, garde un mockup téléphone ou des écrans simulés.
- Ne garde pas une ancienne section si elle rend la page générique, plate ou incohérente avec le playbook.
- Évite les phrases vagues comme "solution innovante", "boostez votre croissance" ou "transformez votre business".

## Sortie
- Réponds uniquement avec le HTML complet.
- Commence par <!DOCTYPE html> et termine par </html>.
- Pas de markdown, pas d'explication.
