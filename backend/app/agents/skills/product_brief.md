# Skill: Brief produit

## Objectif
Produire un document de reference qui definit clairement le produit, son positionnement, et le MVP a construire. Ce brief sera utilise par tous les agents suivants.

## Structure attendue

### 1. Vision produit
- Pitch en 1 phrase (format : "Pour [cible] qui [probleme], [produit] est un [categorie] qui [benefice cle]")
- Vision a 12 mois
- Vision a 3 ans

### 2. Probleme
- Description detaillee du probleme (avec citations d'utilisateurs si possible)
- Impact du probleme sur la cible (temps, argent, frustration)
- Comment la cible resout le probleme aujourd'hui (alternatives existantes)

### 3. Solution proposee
- Description de la solution en 3-5 points
- Differenciation cle vs les alternatives
- USP (Unique Selling Proposition) en 1 phrase

### 4. MVP — Scope minimum
- Liste des fonctionnalites core (5-8 max)
- Ce qui est explicitement HORS scope pour le MVP
- User flow principal (etapes numerotees)

### 5. Positionnement et pricing
- Positionnement vs concurrents (matrice 2x2 si pertinent)
- Strategie de prix recommandee avec justification
- Proposition de valeur par tier (si applicable)

### 6. Metriques de succes
- North Star Metric
- 3-5 KPIs de lancement
- Objectifs a 30, 60, 90 jours

## Regles
- Base-toi sur l'etude de marche precedente si disponible dans le contexte
- Sois specifique au produit, pas de templates generiques
- Chaque section doit etre actionnable
